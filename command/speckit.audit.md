# Spec-Kit Audit Command

## Purpose
Executes full constitutional compliance audit, validates SDD artifacts, verifies quality gates, and ensures security standards compliance across all project documentation and implementation.

## Workflow

### Step 1: Comprehensive Audit Preparation
- Initialize audit framework and scope definition
- Catalog all project artifacts and documentation
- Establish audit criteria and success metrics
- Set up audit trail and documentation system
- Prepare audit reporting templates

### Step 2: Constitutional Compliance Audit
- Execute comprehensive constitutional standards review
- Validate all artifacts against governance principles
- Check alignment with OpenCode constitutional framework
- Assess decision-making process compliance
- Verify stakeholder engagement requirements

### Step 3: SDD Artifact Validation
- Perform complete SDD artifact audit
- Validate design documentation completeness
- Check technical specification accuracy
- Verify architecture diagram compliance
- Assess interface and integration documentation

### Step 4: Quality Gates Verification
- Execute all quality gate validations
- Check performance criteria compliance
- Validate testing and validation procedures
- Assess deployment readiness requirements
- Verify maintenance and support planning

### Step 5: Security Standards Compliance
- Conduct comprehensive security audit
- Validate security requirements implementation
- Check vulnerability assessment procedures
- Verify security testing compliance
- Assess incident response planning

### Step 6: Audit Reporting & Recommendations
- Generate comprehensive audit report
- Document compliance status across all areas
- Identify critical gaps and violations
- Provide prioritized remediation plan
- Create audit dashboard and metrics summary

## Input Requirements
- Complete project documentation set
- Constitutional standards and guidelines
- SDD templates and validation criteria
- Quality assurance frameworks
- Security standards and checklists
- Audit reporting templates

## Output Artifacts
- Comprehensive constitutional audit report
- SDD compliance validation summary
- Quality gates verification results
- Security standards compliance assessment
- Detailed remediation roadmap
- Executive audit summary and dashboard

## Audit Framework
- **Constitutional Compliance**: 100% alignment with OpenCode constitution
- **SDD Standards**: Complete design documentation compliance
- **Quality Assurance**: All quality gates passed and validated
- **Security Standards**: Full security requirements compliance
- **Documentation Integrity**: Complete and accurate documentation
- **Process Adherence**: All processes follow established procedures

## Audit Criteria
1. **Governance Compliance**: Constitutional principles properly implemented
2. **Technical Standards**: SDD artifacts meet industry and project standards
3. **Quality Metrics**: All quality gates and metrics satisfied
4. **Security Requirements**: Security standards fully addressed and validated
5. **Documentation Quality**: Complete, accurate, and consistent documentation
6. **Process Integrity**: All processes follow established procedures and guidelines

## Integration & Follow-up
- Coordination with remediation planning
- Integration with continuous improvement processes
- Connection to governance review cycles
